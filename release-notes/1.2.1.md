- Double-clicking or opening a saved `.lmd` file from the OS now launches LMDmapper directly into that project instead of requiring a second manual load from inside the app.
- Overview image-alignment value fields now apply typed scale/offset values only on `Enter` or blur, instead of rewriting them while the user is typing.
- Overview image alignment now includes an `Import from` selector that copies scale/offset values from another cryosection in the same session.
- Right-clicking without dragging in Coordinates now opens a manual-coordinate assignment modal with prefilled stage coordinates and a metadata-row selector, and assigned rows are labeled `Manual coordinate` in Metadata.
- Coordinates and Overview now show a default coordinate grid before any LIF/CSV files are loaded, using an initial vertical range of roughly `10,000–50,000` and auto-fitting the horizontal span from `0` to match the viewer aspect ratio.
- Manual coordinate rows now stay in warning state until the user confirms them.
- The manual-coordinate assignment popup now includes a `Size` field with a default value of `5000`.
- Manual coordinate assignments now carry their `Size` into Metadata correctly, keep `Manual coordinate` visible in the `Images` column, and keep the warning-state validation/unlink actions available.
- Right-clicking in Coordinates now keeps the viewer stable and uses the original click location when opening the manual-coordinate assignment popup.
- Manual-coordinate status popups now avoid image-related wording, and removing a manual coordinate also removes its stored `Size`.
- Coordinates now provides a `Reuse from` control to copy the loaded LIF and CSV sources from another cryosection without importing the files again.
- The Coordinates `Reuse from` controls now appear on their own row below the import buttons and use the app-styled dark cryosection dropdown.
- CSV routing for cryosections assigned across multiple plate segments now explicitly consumes source columns in order (`1–6`, then `7–12`) and warns when a cryosection is assigned beyond the 12 available CSV columns.
