- Add change notes for this release as they are implemented.
- Keep entries concise and user-facing.
- Shared CSV reuse across cryosections now consumes source columns in group order, so later cryosections that reuse the same CSV/LIF set pick up the next `1–6` / `7–12` block instead of repeating the first half, and older shared-source projects now try to repair those mappings on load when the original CSV files are still reachable.
- Coordinates now includes a `Detach sources` action that clears imported LIF/CSV state and cached coordinates for the active cryosection, making it easier to force a clean reimport when shared-source mappings need to be rebuilt.
- CSV source columns are now resolved before plate assignment, so the second cryosection in a shared-source pair actually receives source columns `7–12` instead of replaying the first six columns through the plate mapper.
